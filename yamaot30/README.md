# ימאות 30 — אפליקציית לימוד

קובץ HTML יחיד, פועל מקומית, לא דורש שרת או אינטרנט אחרי הטעינה.

## מה כלול

- **362 שאלות** מתוך מאגר israelsails.com (yamaot30), עם כל התשובות הנכונות.
- **100 תמונות סימונים חתוכות** (Sign images), מוטמעות בקובץ — אין צורך ב-PDF חיצוני, אין צורך לגלול בתוך PDF, כל תמונה מוצגת ישירות נקייה ובהירה.
- **PDF הסימונים הרשמי** מוטמע גם הוא כ-fallback (כפתור "פתיחת PDF הסימונים" אם בכל זאת רוצים את הגיליון המלא).
- **הסברים מפורטים מובנים ל-54 תמונות** (40 מאומתות, 14 דורשות בדיקה ידנית). שאר התמונות מסומנות "דורש בדיקה ידנית" וניתנות לעריכה בתוך האפליקציה.
- **5 מצבי לימוד:** סבב חכם · טעויות וניחושים · שאלות עם תמונה · מבחן 50 · מאגר חיפוש.
- **חזרה מרווחת** (Spaced repetition): סולם 1→3→7→14→30 ימים. תשובה נכונה מקדמת, שגויה מאפסת, ניחוש קופץ ל-1.
- **גיבוי וייבוא JSON** ידני.
- **סנכרון אוטומטי בין מכשירים** דרך Vercel KV (Upstash Redis). אוטומטי מהרגע שפותחים את ה-URL בכל מכשיר — ללא הגדרות.
- **עוגני עמוד ל-PDF** (page anchors / "מפות"): כל תמונה יכולה לקבל קישור לעמוד ספציפי ב-PDF הסימונים עם זום.
- **ערכת נושא בהירה** כברירת מחדל, עם החלפה לכהה (כפתור 🌙/☀ בכותרת).

## איך משתמשים

### במחשב
1. לחץ פעמיים על `yamaot30-study.html` — נפתח בדפדפן.
2. ההתקדמות נשמרת אוטומטית ב-localStorage של הדפדפן.

### בטלפון
1. שלח את `yamaot30-study.html` לטלפון (Drive / WhatsApp / מייל / USB).
2. שמור על הטלפון, פתח עם דפדפן (Chrome / Safari).
3. (אופציונלי) הוסף ל-Home Screen להפעלה כמו אפליקציה.

## סנכרון בין מחשב וטלפון

### אפשרות א — סנכרון אוטומטי דרך Vercel (ברירת מחדל, ללא הגדרות)

האפליקציה פרוסה בכתובת `https://yamaot30.vercel.app`. כל מכשיר שנכנס לכתובת הזו מסנכרן אוטומטית:

1. **כניסה ראשונה במכשיר חדש:** האפליקציה בודקת את השרת. אם בשרת יש מצב מתקדם יותר — מורידה אותו. אם המכשיר הזה מתקדם יותר — דוחפת לשרת.
2. **כל תשובה/עריכה לאחר מכן:** נדחפת לשרת אוטומטית תוך 2 שניות (debounced).
3. **התנגשות (etag mismatch):** קורית רק אם שני מכשירים שומרים בו-זמנית. סטטוס הסנכרון יציג "התנגשות — אישור ידני נדרש". פתח את "סנכרן עכשיו" ובחר Push (לדרוס את השרת) או Pull (להוריד מהשרת). אישור ידני אינו נדרש בזרימה השקטה כדי לא להפריע באמצע לימוד.

הסנכרון מבוסס על Upstash Redis (תוכנית חינמית) המחוברת לפרויקט Vercel. כתובת ה-API:
- `GET /api/health` — בדיקת חיים.
- `GET /api/study/state/<APP_ID>` — קריאת מצב נוכחי.
- `PUT /api/study/state/<APP_ID>` — שמירה (תומך ב-`expectedEtag` ל-optimistic concurrency).

### אפשרות ב — JSON ידני (תמיד זמינה כגיבוי)
1. בצד שיש לך התקדמות: "יצא JSON".
2. העבר את הקובץ למכשיר השני.
3. שם: "יבא JSON" → אישור.

### אפשרות ג — סנכרון אוטומטי דרך OpenClaw dashboard (לוקאלי, ללא אינטרנט)
דורש את OpenClaw control-dashboard רץ במחשב (port 7777 ב-loopback, 7787 ב-Tailscale). מפורט בגרסה הקודמת של הקובץ הזה. כיום מומלץ להשתמש באופציה א'.

## העלאת גזירה אישית של תמונה

100 התמונות כבר חתוכות וטעונות. אם תרצה להחליף תמונה ספציפית בגרסה חתוכה משלך:
1. עבור לשאלה שמפנה לתמונה.
2. לחץ "📷 החלף גזירה" → בחר PNG/JPG.
3. הגזירה האישית מחליפה את המוטמעת. "🗑 מחק גזירה אישית" מחזיר.

## עריכת הסברים

לחץ "עריכת/הוספת הסבר" בתחתית פאנל ההסבר. ערוך 7 שדות:
1. מסקנה רשמית
2. מה רואים
3. פירוט לפי אורות (שורה לפריט)
4. כיוון יחסי
5. סיכום פשוט
6. טעות נפוצה
7. שיטת זיכרון

סמן ☑ "מאומת" כדי להסיר את התג "דורש בדיקה ידנית".

ההסברים נשמרים ב-localStorage ועוברים בייצוא/ייבוא JSON וגם בסנכרון האוטומטי.

## פריסה ל-Vercel

הפרויקט מחובר ל-GitHub `Eladtz2025/yamaot30`. Vercel דוחפת אוטומטית ל-`yamaot30.vercel.app` בכל push ל-main.

לעדכון מהיר ללא git:
1. דחוף את `vercel/public/yamaot30-study.html` המעודכן לשורש ה-repo ב-GitHub (drag-drop ב-web UI ל-upload).
2. Vercel מזהה ובונה תוך ~10 שניות.
3. אם הפריסה לא מקבלת קידום אוטומטי לפרודקשן (למשל אחרי rollback), פתח את ה-deployment ב-Vercel → "..." → Promote.

תלויות סביבה (מוזרקות אוטומטית ע"י אינטגרציית Upstash):
- `KV_REST_API_URL`
- `KV_REST_API_TOKEN`
- `KV_REST_API_READ_ONLY_TOKEN`
- `KV_URL`, `KV_REDIS_URL`
