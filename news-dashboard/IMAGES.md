# Article Images Plan

## Goal
Keep cards visually compact and easy to scan. Do not use article images by default if they make stories harder to distinguish.

## Preferred order
1. No image by default
2. Minimal visual header only if needed for separation
3. Use article images only when explicitly desired and when they improve clarity

## UI rule
- Whole card should be clickable.
- Do not show a separate visible "קישור" label inside the card.
- Keep story cards compact.
- Prefer clean text separation over decorative imagery.

## Runtime note
Image extraction is source-dependent and may remain partial in blocked runtimes.
