# Deep Scan Rule

## Rule
For each strong category source, attempt to review multiple recent same-day items before concluding the category is thin.

## Practical target
- preferred: ~10 recent items per usable source when extraction allows
- minimum: enough breadth to justify top 5 selection

## Category impact
- Technology: must not stop at 2 if more usable sources exist
- News: same-day scan first, fallback later
- Crypto: same-day scan first, then weekly fallback if needed
- Hapoel: fixture-first + multiple sports sources before declaring weak retrieval
